title: Real-time Path-Explosive Monitoring
---

**Background:** The path-explosive framework is inherently retrospective, as it requires completed window patterns to identify and categorize explosive episodes. This constraint limits the framework's utility for real-time monitoring and early warning systems.

**Question / Future Work:** Developing a sequential monitoring version of the framework that applies window detection and provisional gate statistics as new data points arrive is necessary for practical application. This would involve creating a two-state system (early warning and confirmed classification) that accounts for uncertainty.

**Why It Matters:** Extending the framework to support real-time detection is critical for policy intervention and infrastructure capacity planning, where decisions are time-sensitive.

**Evidence:** The framework is primarily a retrospective tool... A natural extension is sequential monitoring: applying the window detector and provisional gate statistics as new annual observations arrive.
