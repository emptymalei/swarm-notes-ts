title: Formalizing Threat Models for Forecasting
---

**Background:** The ISA/IEC 62443 standards series defines security maturity levels, but current open-source forecasting libraries lack established, formal threat models for specific network-mediated components.

**Question / Future Work:** Achieving higher security maturity levels (e.g., Defined/ML3) in forecasting software requires the formalization and documentation of a comprehensive threat model that can be standardized and shared across different open-source libraries to facilitate broader security auditing and interoperability in safety-critical systems.

**Why It Matters:** Formalizing threat models is a prerequisite for advancing from Managed to Defined maturity levels under industrial cybersecurity standards, which is essential for auditability in critical infrastructure.

**Evidence:** The maturity level ML 3 Defined requires a formal threat model, which is listed among the future work in Chapter 8.
