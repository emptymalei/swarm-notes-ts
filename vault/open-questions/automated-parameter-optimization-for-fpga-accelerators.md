title: Automated Parameter Selection Framework
---

**Background:** Automating the selection of hardware-specific design parameters allows for the optimization of machine learning accelerators across varying computational contexts and resource constraints.

**Question / Future Work:** Developing methods to automatically determine the optimal configuration of architectural parameters—such as resource allocation (DSPs vs. LUTs), quantization schemes, and activation function implementations—for specific LSTM model architectures and target hardware constraints remains a significant challenge. Future efforts aim to integrate this parameter selection into end-to-end design flows to maximize efficiency for complex deep learning models on resource-constrained embedded FPGAs.

**Why It Matters:** Currently, manual configuration of hardware accelerators for different deployment scenarios is time-consuming and non-optimal, which hinders the widespread adoption of customized FPGA-based acceleration in resource-limited embedded domains.
